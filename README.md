# web-main-archive

本仓库用于归档和发布多个独立的 HTML 页面。  
这些页面可能来自不同项目、不同时期，或不同用途，统一存放在此以便索引和访问。

## 项目简介

`web-main-archive` 是一个静态 HTML 文件的集合仓库。  
它不服务于某个特定站点或域名，而是作为多个独立页面的统一归档空间。  
你可以将其视为一个“页面仓库”，用于存放那些需要保留或展示的 HTML 文件。

## 目录说明

```
.
├── index.html          # 仓库首页（可选）
├── pages/              # 归档的 HTML 页面文件夹
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 公共资源（样式、脚本、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/` 目录下存放各个独立的 HTML 页面。
- `assets/` 目录用于存放页面可能引用的公共资源，如样式表、脚本文件或图片。
- 每个页面应尽量保持自包含，或明确引用公共资源路径。

## 页面归档说明

- 所有页面均为独立 HTML 文件，彼此之间没有强依赖关系。
- 页面可能包含自己的样式和脚本，也可能引用 `assets/` 下的公共资源。
- 归档的页面可能来自不同时间、不同项目，因此风格和内容可能差异较大。
- 每个页面应当有一个清晰的标题或描述，以便于识别。

## 维护说明

- 新增页面请放入 `pages/` 目录，并保持命名简洁、无空格。
- 若页面需要引用公共资源，请确保资源路径正确。
- 不建议在页面中嵌入指向外部服务的动态内容（如 API 调用），以保持归档的稳定性。
- 如需删除或替换页面，请谨慎操作，并更新相关索引（如有）。

## 使用方式

1. 克隆或下载本仓库。
2. 直接在浏览器中打开 `pages/` 下的 HTML 文件即可查看。
3. 也可将整个仓库部署到静态托管服务（如 GitHub Pages），以在线浏览。

本仓库的内容遵循相应的许可协议。
